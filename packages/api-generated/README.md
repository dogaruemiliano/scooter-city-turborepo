# `@repo/api-generated`

> **⚠️ DO NOT EDIT FILES IN `src/` BY HAND.** Everything except `mutator.ts` is auto-generated by Orval from the API's OpenAPI spec.

Typed fetch client + request/response types derived from `apps/api`'s controllers. Regenerated end-to-end by:

```bash
pnpm gen
```

at the repo root. That pipeline:

1. Refreshes `apps/api/.env.example` from the zod env schema (`pnpm gen:env`).
2. Builds `@repo/api-shared` so the compiled API can load it.
3. Builds `apps/api` (`prisma generate` + `nest build`).
4. Boots `apps/api/dist/src/main.js --spec-only` and writes the OpenAPI 3 document to `openapi.json` at the repo root.
5. Runs Orval (`orval --config ./orval.config.ts`) which reads `openapi.json` and writes the generated client into `src/`.

## What's hand-maintained

| File                                                              | Owner                                                                                                                                             |
| ----------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| `src/mutator.ts`                                                  | Hand-written. The custom `fetch` wrapper Orval calls for every operation. Sets `credentials: "include"` so HttpOnly cookies travel automatically. |
| `package.json`, `tsconfig.json`, `eslint.config.mjs`, this README | Hand-written.                                                                                                                                     |
| Everything else under `src/`                                      | Generated. Regenerate after API controller changes.                                                                                               |

## CI / pre-commit drift guard

A `pnpm gen` drift check in CI is a documented follow-up: it'll re-run the pipeline and fail the build if `src/` differs from what's committed.

## Consumers

- `apps/web` imports operations from this package's `src/index.ts`.
- `apps/mobile` (future) will too.
- `apps/api` is the _source_, never a consumer.
